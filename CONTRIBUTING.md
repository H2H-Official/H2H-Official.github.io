# Contributing to H2H

      We are building a decentralized privacy stack. We don't accept low-effort code. To contribute, follow these standards:

      1. **Fork the repo.**
      2. **Implement your patch** in a new branch (e.g., `feature/routing-patch`).
      3. **Add Tests:** Every new feature MUST include unit tests in the `/tests` directory. We will not merge code that isn't tested.
      4. **Pull Request:** Open a PR with a technical summary of the fix.
      5. **Code Review:** We will review your code for security, performance, and anonymity.

      Only verified, secure, and performant code will be merged into the H2H core.
